# Signatate_練習問題「お弁当の需要予測」
Signateの練習問題「お弁当の需要予測」について、需要数の予測モデル(重回帰モデル)を構築しました。
評価指標はRMSEで、最終スコアは 11 でした。

## プロジェクト概要
- タスク: 日付・曜日・天気・メニュー情報などから、お弁当の需要数を予測する回帰問題
- モデル: scikit-learn を用いた重回帰モデル
- データ処理:
  - EDA による分布・欠損の確認
  - カテゴリ変数（曜日・天気など）のダミー変数化
  - kcal・気温・降水量など数値特徴量の標準化
- 評価: ホールドアウト検証による RMSE の算出

## 全体方針      
1.探索的データ分析(EDA)を行う。
2.各特徴量に4つの尺度(名義、順序、間隔、比例)を当てる。
3.特徴量エンジニアリング⓵:特徴量改善を行う。
4.特徴量エンジニアリング⓶:特徴量作成を行う。
5.特徴量エンジニアリング⓷:特徴量選択を行う。
6.訓練を通じ、特徴量エンジニアリングとモデル選択が正しいかを評価する。
7.テストデータの特徴量に同じ処理を施し、予測する。
8.予測結果をcsvへ出力する。

## 使用技術・ライブラリ
- 言語: Python
- ライブラリ:
  - pandas (データ整形・集計・前処理)
  - scikit-learn (前処理・モデル学習・評価)

## 学習の狙い
- EDA から特徴量エンジニアリング、モデル構築、評価、提出ファイル作成までの一連の流れを自力で実装すること  
- 曜日・天気などのカテゴリ情報や、kcal・気温・降水量などの数値情報をどのように特徴量として扱うかを実践的に学ぶこと
